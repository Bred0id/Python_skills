# Контест о списках и словарях

Для сдачи задачи под номером N нужно перейти в ветку task{N} и сделать коммит с кодом.

Мы создаём приложение "Умный холодильник" - оно будет хранить информацию о пользователях в семье, предупреждать о сроках хранения продуктов, предлагать рецепты из того, что есть, балансировать питание.

## Задача 1. Хитрый пароль
Для начала нужно реализовать функцию проверки пароля пользователя на пригодность. Напишите функцию check_password в task1.py, которая на вход будет принимать никнейм пользователя (username) и предложенный пароль (password), а на выходе - True, если он подходит под условия надёжности пароля и False если иначе.
Правила хорошего пароля:

- Его длина - хотя бы 5 символов

- В пароле должна встречаться хотя бы одна цифра

- В пароле должны быть как заглавные, так и строчные буквы

- Сумма всех цифр в пароле должна равняться 25

- В пароле должно встречаться название хотя бы одного из месяцев (january, february, march, april, may, june, july, august, september, october, november, december) - регистр не имеет значения

- Имя пользователя вне зависимости от регистра НЕ должно быть частью пароля.


## Задача 2. Подсчёт калорий
Напишите функцию count_calories в task2.py, которая будет считать количество калорий со всех приёмов пищи. На вход ей поступает произвольное число (может быть, 0) элементов (!!не список), каждый из которых является числом калорий, потреблённых из конкретного продукта. На выходе нужна сумма калорий.

## Задача 3. Аллергии
У вас есть список съеденного за день в следующем формате:

```
all_foods = [{"name": "Печенье",

    "ingredients": ["Мука", "Соль", "Сахар", "Шоколад"]},

    {"name": "Кофе с сиропом",

    "ingredients": ["Молоко", "Кофе", "Шоколад"]},

    {"name": "Салат 'Цезарь'",

    "ingredients": ["Курица", "Салат", "Пармезан", "Помидоры", "Майонез"]}

]
```



Напишите функцию find_allergies, которой на вход приходит подобный список all_foods и произвольное количество аллергенов. Нужно вернуть список названий блюд, в которых содержались аллергены.

Внимание! Воспользуйтесь comprehensions в этом задании.

## Задача 4. Любимый продукт
У вас есть список того, что съел пользователь, и его оценки в следующем формате:

```
all_foods = [ {"name": "Печенье",

    "ingredients": ["Мука", "Яйцо", "Соль", "Сахар", "Шоколад"],

    "rating": 5},

    {"name": "Кофе с сиропом",

    "ingredients": ["Молоко", "Кофе", "Шоколад"],

    "rating": 4},

    {"name": "Салат 'Цезарь'",

    "ingredients": ["Курица", "Салат", "Пармезан", "Помидоры", "Майонез"],

    "rating": 3},

    {"name": "Паста Карбонара'",

    "ingredients": ["Спагетти", "Яйцо", "Бекон", "Перец"],

    "rating": 2},

    {"name": "Клубника в шоколаде",

    "ingredients": ["Клубника", "Шоколад"],

    "rating": 5},

    {"name": "Паста Карбонара'",

    "ingredients": ["Спагетти", "Яйцо", "Бекон", "Перец"],

    "rating": 2}

]
```



Напишите функцию find_top_k_ingredients, которая на вход принимает подобный список, числа k и n, а на выходе выводит топ k ингредиентов, которые нравятся пользователю больше всего и встречаются не менее n раз, в порядке возрастания. В случае, если есть несколько продуктов с одинаковым значениями средней оценки, выбираем их по алфавитному порядку в порядке возрастания.

Формула оценки ингредиента следующая - {Сумма оценок за блюда, которые ел пользователь, в которых есть этот ингредиент} / {Количество оценок, которые ставил пользователь этим блюдам}
